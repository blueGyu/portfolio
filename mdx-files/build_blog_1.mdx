## 블로그 제작기 1편 - Next.js, tailwindcss로 블로그 한 입

다시 IT회사에 취업하기 위해 뭐부터 해야할까 고민하다가 개인 블로그 겸 포트폴리오 사이트를 제작하며 React, Typescript를 연습하기로 했다.
찾다보니 마크다운을 이용해서 게시글을 작성하는 방법이 있다는 것을 알게 되어 해당 방법으로 사이트를 제작하기로 했다.

### 1. 마크다운과 정적 사이트

먼저, 마크다운 파일을 페이지에 표현하는 것부터 시작하였다.
웹 유시보수는 해봤지만, 맨땅에 새로 만드는 것은 처음이라 다른 사람이 한 자료를 참고하였다. 참고자료는 아래 이미지를 확인하시면 됩니다.

기본적인 흐름은 Nextjs의 Dynamic Router를 이용해 일치하는 마크다운 파일의 metadata 부분과 content를 가져와서 화면에 보여주는 것이다.
여기까지만 하면 정적 사이트라고 할 수 없지. generateStaticParams를 이용하면 빌드 시에 Dynamic Router Segment를 정적인 경로로 생성해줍니다.

Next.js 공식 홈페이지에서 generateStaticParams를 확인하고 싶으시다면 [여기](https://nextjs.org/docs/app/api-reference/functions/generate-static-params)를 클릭해주세요.

영상과는 다르게 <Library library="markdown-to-jsx" /> 대신에
mdx을 사용하여 마크다운 중간에 컴포넌트를 삽입 또는 jsx 문법을 사용하기 위해 <Library library="next-mdx-remote" /> 라이브러리를 적용하였습니다.

Next.js 공식 홈페이지에서 MDX 파일을 적용하는 방법을 참고하였습니다. 방법을 확인하고 싶다면 [여기](https://nextjs.org/docs/app/building-your-application/configuring/mdx)를 클릭해주세요.

<Library library="next-mdx-remote" /> 라이브러리의 공식 페이지는 [여기](https://github.com/hashicorp/next-mdx-remote#react-server-components-rsc--nextjs-app-directory-support)를
클릭해주세요.

<Code example={`import { MDXRemote } from "next-mdx-remote/rsc";
import PostImage from "@/components/mdx/PostImage";
import Library from "@/components/mdx/Library";

const components = {
  a: (props: ComponentProps<"a">) => (
    <a {...props} target="blank">
      {props.children}
    </a>
  ),
  PostImage,
  Library,
  Code
};

export default function PostPage({ params }: { params: { category: string; postname: string } }) {
  const { postname } = params;
  const post = getPostContent(postname);
  return (
    <div className="flex justify-center py-5">
      <article className="prose max-w-[840px]">
        <MDXRemote source={post.content} components={components} />
      </article>
    </div>
  );
}`
}
/>
마크다운 중간에 jsx 문법을 적용하기 위해서는 몇 가지 작업이 필요합니다.
위의 코드처럼 MDXRemote 태그의 속성에 components를 추가하면, mdx 파일의 마크다운 중간에 추가한 컴포넌트 또는 각 태그를 커스텀하여 사용할 수 있다.

### 2. 마크다운에 스타일 한 스푼

동영상을 참고하여 사이트를 만들었을 경우, 게시글에 디자인이 적용이 되어있지 않다.
찾다보니 tailwindcss의 공식 플러그인, <Library library="tailwindcss/typography" />로 간편하게 스타일을 적용할 수 있었다.

  <div className="flex flex-col justify-center items-center mt-3">
    <span className="text-sm">tailwind에서 확인할 수 있는 글</span>
  </div>
</PostImage>

<Library library="tailwindcss/typography" /> 공식 페이지로 이동을 원하신다면 [여기](https://github.com/tailwindlabs/tailwindcss-typography)를
클릭해주세요.

장점은 쉽고 빠르게 마크다운에서 JSX로 변환된 요소에 스타일을 적용할 수 있고 사용 방법이 tailwindcss와 같다. 단점은 커스텀하려면 생각보다 수정해야할 부분이 많을 수 있다.

### 정리

위 여러 사이트를 참고하며 Next.js와 tailwindcss를 사용하여 정적 블로그의 뼈대를 만들었다. 두 스택을 사용하며 느낀 점은 생각보다 빠르게 웹 사이트를 구축할 수 있었다.
아직 깊이 사용해보지는 못했지만, 해당 스택을 사용하는 회사에 취업했으면 좋겠다.
